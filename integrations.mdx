## Webhooks

Set up webhooks to receive instant updates on compliance status changes, such as prescription approvals or audit flags.

### Setup Steps

<Steps>
  <Step title="Create Webhook" icon="plus">
    In your C3 Access dashboard, go to Integrations > Webhooks > New Webhook.
  </Step>
  <Step title="Add Endpoint URL" icon="link">
    Enter your server URL, e.g., `https://your-pharmacy-server.com/webhook`.
  </Step>
  <Step title="Verify and Activate" icon="check-circle">
    C3 Access sends a verification challenge. Respond with HTTP 200.
  </Step>
</Steps>

### Webhook Payload Example

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  app.post('/webhook', (req, res) => {
    const { event, data } = req.body;
    if (event === 'compliance.updated') {
      console.log(`Patient ${data.patientId} status: ${data.status}`);
    }
    res.status(200).send('OK');
  });
  ```
  ```python
  from flask import Flask, request

  app = Flask(__name__)

  @app.route('/webhook', methods=['POST'])
  def webhook():
      data = request.json
      event = data['event']
      if event == 'compliance.updated':
          print(f"Patient {data['data']['patientId']} status: {data['data']['status']}")
      return 'OK', 200
  ```
</CodeGroup>

## API Endpoints Overview

Use our REST API for programmatic access. Base URL: `https://api.example.com/v1`.

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://api.example.com/v1/compliance/verify \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "patientId": "12345",
      "prescription": {
        "drug": "oxycodone",
        "quantity": 30
      }
    }'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/compliance/verify', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      patientId: '12345',
      prescription: {
        drug: 'oxycodone',
        quantity: 30
      }
    })
  });
  ```
</Request>

<Response tabs="200,400">
  ```json
  {
    "status": "approved",
    "patientId": "12345",
    "restrictions": [],
    "verifiedAt": "2024-01-15T10:30:00Z"
  }
  ```
  ```json
  {
    "error": "invalid_drug_schedule",
    "details": "DEA Schedule II requires special handling"
  }
  ```
</Response>

<ResponseField name="status" field-type="string" required="true">
  Compliance decision: `approved`, `rejected`, or `pending`.
</ResponseField>
